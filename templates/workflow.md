# Workflow: {{project_name}}

> 由 micro-harness 自动初始化
> 当前阶段: 1 — 需求澄清

---

## 阶段 1：需求澄清 🎯

**目标**：明确要做什么，确认输入和输出

- [ ] 理解项目/任务的核心目标
- [ ] 确认输入条件（文件、环境、依赖等）
- [ ] 确认验收标准（怎样才算做完）
- [ ] 如果需求不明确 → 先问用户再动手

> ✅ 完成本阶段后 → 进入阶段 2

---

## 阶段 2：规划 📋

**目标**：把任务拆成可执行的步骤

- [ ] 拆解任务步骤（每步一个可验证的成果）
- [ ] 评估每步复杂度（简单/中等/复杂）
- [ ] 标注依赖关系（哪步必须先做）
- [ ] 如果步骤 > 5 → 考虑分多个 task 或合并相似步骤

> ✅ 完成本阶段后 → 进入阶段 3
> 📝 拆解结果写入 `tasks/current.md`

---

## 阶段 3：执行 ⚡

**目标**：按规划逐步执行

- [ ] 按 task 顺序逐步执行
- [ ] 每完成一步 → 更新 `tasks/current.md`
- [ ] 遇到问题 → 记入 `bugs.md`
- [ ] 遇到重大决策 → 在 `checkpoints/` 创建还原点
- [ ] 遵守 `boundaries.md` 中的重试限制

> 🔄 当前进行中

---

## 阶段 4：验证 ✅

**目标**：确认结果正确

- [ ] 检查输出是否符合预期
- [ ] 验证没有遗留问题
- [ ] 跑一遍关键测试/检查
- [ ] 对照验收标准逐条确认

> ✅ 完成本阶段后 → 进入阶段 5

---

## 阶段 5：收尾 🏁

**目标**：总结经验，清理现场

- [ ] 更新 `journal.md`（做了什么、遇到什么、学了什么）
- [ ] 清理临时文件（下载的安装包等）
- [ ] 告知用户项目完成

> ✅ 完成后 → 进入阶段 6（自进化）

---

## 阶段 6：自进化 🧬

**目标**：反哺技能，让下次更聪明

执行技能自进化流程（详见 micro-harness 技能说明）：

- [ ] 分析本次项目的 bugs/checkpoints/journal
- [ ] 提取可复用的经验模式
- [ ] 如果有 2+ 次重复模式 → 更新 skill 模板
- [ ] 记录进化日志

> 每一次项目，都在让这个技能变强。
